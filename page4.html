Proceso Web One Page

COMERCIAL:
- Asesor debe llenar el Brief Mínimo Obligatorio
- Si asesor no proporciona información para la página web esta se va aNo Conformidad, tener presente que este tipo de páginas no tiene relevamiento. 

PRODUCCIÓN:
- Gestión de Clientes accede al BMO y elabora la hoja de redacción 
- Envía a diseñador pedido para elaborar la Página Web
- Una vez elaborado el diseño pasa a Etapa QA en Sales Force
- Gestión de Clientes revisa página, compra el dominio y publica la web
- Gestión de clientes configura buzones de correos
- Tiempo aproximado de producción 10 a 15 días.

Plazos de Relevamiento

GESTIÓN DE CLIENTES
- A excepción de la Web EMPRENDE todas las Páginas Web pasan por la etapa de relevamiento
- EL cliente tiene un plazo de 7 días utiles
- En el plazo de 7 días se le deben efectuar un mínimo de 3 llamadas al cliente
- Si Cliente no proporciona más información se elabora hoja de redacción con información del BMO
- Si Cliente solicita más plazo para entregar información, se envía a etapa Aplazado por el Cliente un máximo de 10 días, sino cumple se procede como paso anterior.
- Si Cliente durante relevamiento manifiesta que no desea la Web se envía a No Conformidad

Cliente no aprueba Publicación

GESTIÓN DE CLIENTES 
- Si Cliente no aprueba publicación se le da un plazo máximo de 10 días calendario
- Si Cliente insiste en no publicar página web se envía a No Conformidad para que asesor apoye a conseguir la aprobación. 

Cliente en Aplazado no completa información

GESTIÓN DE CLIENTES
- Si Cliente no proporciona más información se elabora hoja de redacción con información del BMO
- Si la información del BMO es insuficiente se envía a No Conformidad.

Dominios

Preguntas frecuentes

Estados un Dominio según: www.whois.com

Un dominio registrado puede tener asignados diferentes estados, que hacen referencia a la situación actual del dominio (ciclo de vida) y qué acciones se pueden realizar con el mismo.

Entre los estados de un dominio te puedes encontrar los siguientes:

OK: El dominio se encuentra activo. Es el estado habitual de un dominio registrado, no caducado.

clientTransferProhibited: El dominio se encuentra activo, pero bloqueado. En este estado el dominio no puede ser transferido a otro registrador.

clientDeleteProhibited: El dominio se encuentra activo. Es un estado que impide solicitar la eliminación del dominio. Suele aparecer junto a clientTransferProhibited y clientUpdateProhibited.

clientUpdateProhibited: El dominio se encuentra activo. Es un estado que impide solicitar la actualización del dominio. Suele aparecer junto a clientTransferProhibited y clientDeleteProhibited.

clientHold: El dominio se encuentra caducado e inactivo. Dentro del ciclo de vida este estado corresponde al Periodo de Gracia. En este estado es posible renovar el dominio a precio normal, para que vuelva a estar activo.

redemptionPeriod: El dominio se encuentra caducado e inactivo. Dentro del ciclo de vida este estado corresponde al Periodo de Castigo. En este estado es posible renovar el dominio a mayor precio, para que vuelva a estar activo.

pendingDelete: El dominio se encuentra al final de su ciclo de vida, pendiente de ser eliminado. Dentro del ciclo de vida este estado corresponde al Periodo de Eliminación. En este estado el dominio ya no puede ser renovado, por lo que sólo queda esperar a que se libere.

Qué son los DNS?

DNS - Sistema de Nombres de Dominio
El Sistema de Nombres de Dominio (DNS) ayuda a los usuarios a ubicarse en Internet. Cada computadora en Internet tiene una dirección única -comparable a un número telefónico- que consiste en una secuencia numérica bastante complicada. Esa dirección se denomina "dirección de IP" (“IP” significa "Protocolo de Internet"). Las direcciones de IP son difíciles de recordar. El DNS hace que sea más fácil usar Internet, ya que permite visualizar una cadena de letras - el nombre de dominio - que resulta más familiar que la dirección arcana de IP. Entonces, en lugar de ingresar 207.151.159.3, usted puede ingresar www.internic.net. Esto es un recurso nemotécnico que hace que las direcciones sean más fáciles de recordar.

Qué es la IP?

IP- Protocolo de Internet
Protocolo de comunicaciones que subyace a Internet. El IP permite que grandes redes de computadoras geográficamente diversas se comuniquen entre sí en forma rápida y económica a través de una serie de enlaces físicos. Una dirección de Protocolo de Internet es la dirección numérica mediante la cual se identifica una ubicación en Internet. Las computadoras en Internet utilizan las direcciones IP para enviar tráfico y establecer conexiones entre sí; las personas suelen usar los nombres familiares que son posibles gracias al Sistema de Nombres de Dominio.

Qué es WHOIS?

WHOIS
Protocolo de WHOIS (se pronuncia "who is" - en inglés – y no es un acrónimo). WHOIS es un protocolo de Internet utilizado para consultar bases de datos y obtener información sobre la registración de un nombre de dominio (o dirección IP). La especificación inicial del protocolo de WHOIS se publicó en el documento RFC 954 en1985. La especificación actual figura en el documento RFC 3912. En los acuerdos de gTLD de la ICANN, se requiere que tanto registros como registradores ofrezcan una página web interactiva y un servicio de Puerto 43 para permitir el acceso público y gratuito a los datos de los nombres registrados. Esos datos son comúnmente conocidos como “datos de WHOIS” e incluyen elementos como las fechas de creación y vencimiento de las registraciones de nombres de dominio, servidores de nombre, información de contacto del registratario y contactos administrativos y técnicos designados.

Por lo general, los servicios de WHOIS se utilizan para identificar a los titulares de nombres de dominio con fines comerciales, ya quienes puedan solucionar problemas técnicos relacionados con el dominio registrado.

Qué es la ICANN?

ICANN - Corporación para la Asignación de Nombres y Números en Internet
La Corporación para la Asignación de Nombres y Números en Internet (ICANN) es una corporación internacional sin fines de lucro responsable de la asignación del espacio de direcciones de Protocolo de Internet (IP), la asignación de identificadores de protocolo, la administración del sistema de nombres de dominio genéricos de alto nivel (gTLD) y de dominios de alto nivel con código de país (ccTLD), y las funciones de administración del sistema del servidor raíz. Originalmente, estas funciones eran desempeñadas por la Autoridad de Números Asignados en Internet (IANA) y otras entidades mediante un contrato con el gobierno de los Estados Unidos. Actualmente, la ICANN desempeña las funciones de la IANA. Como asociación privada-pública, la Corporación para la Asignación de Nombres y Números en Internet (ICANN) está dedicada a preservar la estabilidad operativa de Internet; promover la competencia; lograr una amplia representación de las comunidades mundiales de Internet; y elaborar políticas adecuadas a su misión a través de un proceso participativo basado en el consenso.
